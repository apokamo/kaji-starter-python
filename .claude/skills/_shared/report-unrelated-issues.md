# 無関係な問題の報告ルール（共通）

作業中（テスト実行・コードレビュー・ドキュメント確認等）に、自分の変更と無関係な問題を発見した場合の報告手順。

## 対象となる問題

- テスト失敗・エラー
- コードの不具合・デッドコード・ロジック誤り
- ドキュメントの不整合・誤記

## 判定基準

`git diff main...HEAD` の変更差分に含まれるファイル・モジュールと関係がない問題を「無関係」と判定する。

判定に迷う場合は「無関係」と断定せず、既存 Issue 検索の上で報告する（過少報告より過剰報告を選ぶ）。

## 報告フロー

1. 既存 Issue を検索（失敗テスト名・エラーメッセージ・問題の要約で検索）
2. 既存 Issue がある場合 → コメントで追記（再現した旨と環境情報）
3. 既存 Issue がない場合 → 新規 Issue を作成

## 報告時の必須項目

### テスト失敗の場合

- 失敗テスト名（例: `tests/test_foo.py::test_bar`）
- エラーログ（traceback）
- 再現コマンド（例: `pytest tests/test_foo.py::test_bar -v`）
- 発見時のブランチ・コミット

### コード・ドキュメントの問題の場合

- 問題箇所（ファイルパス・行番号・シンボル名）
- 問題の概要と根拠
- 発見時のブランチ・コミット
